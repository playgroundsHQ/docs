# Безпека

fibe.gg забезпечує кілька рівнів захисту вашого облікового запису — від **двофакторної автентифікації** та **ключів безпеки** до **управління сесіями**, **API-ключів** та **вебхуків**. Ця сторінка охоплює все необхідне для забезпечення безпеки вашого облікового запису та інтеграцій.

---

## Двофакторна автентифікація (2FA)

Двофакторна автентифікація додає додатковий рівень безпеки до вашого облікового запису. Навіть якщо хтось отримає ваш пароль, він не зможе отримати доступ до вашого облікового запису без дійсного одноразового коду з вашого додатку автентифікації.

### Увімкнення 2FA

1. Перейдіть до **Profile → Security**
2. Натисніть **Enable Two-Factor Authentication**
3. Відскануйте QR-код вашим бажаним додатком автентифікації (напр., Google Authenticator, Authy, 1Password)
4. Введіть 6-значний код з вашого додатку автентифікації для підтвердження налаштування

Після підтвердження вам буде представлено набір **кодів відновлення**.

### Коди відновлення

Коли ви вмикаєте 2FA, fibe.gg генерує **10 одноразових кодів відновлення**. Ви повинні підтвердити один з цих кодів для завершення процесу налаштування.

Кожен код відновлення може бути використаний лише **один раз**. Після використання код назавжди вважається спожитим.

Якщо у вас закінчуються коди відновлення, ви можете згенерувати новий набір у **Profile → Security**. Перегенерація кодів анулює всі раніше видані коди.

:::caution
Зберігайте свої коди відновлення в безпечному місці — наприклад, у менеджері паролів або захищеному фізичному місці. Якщо ви втратите доступ як до додатку автентифікації, так і до кодів відновлення, ви можете бути заблоковані зі свого облікового запису.
:::

### Вимкнення 2FA

Щоб вимкнути двофакторну автентифікацію, перейдіть до **Profile → Security** та натисніть **Disable 2FA**. Вам потрібно буде ввести дійсний 6-значний код з вашого додатку автентифікації для підтвердження.

### Довіра сесій

Після увімкнення 2FA ваші існуючі сесії потребують повторної верифікації. Доки сесія не буде підтверджена дійсним TOTP-кодом, кодом відновлення або ключем безпеки, вона вважатиметься недовіреною, і вам буде запропоновано підтвердити свою особу перед виконанням чутливих дій.

## Ключі безпеки (WebAuthn)

Ключі безпеки забезпечують стійкий до фішингу спосіб верифікації вашої особи. fibe.gg підтримує **апаратні ключі FIDO2/U2F** (такі як YubiKey), а також **платформні автентифікатори** (такі як Touch ID, Face ID або Windows Hello).

### Додавання ключа безпеки

1. Перейдіть до **Profile → Security**
2. Натисніть **Add Security Key**
3. Дайте ключу впізнавану назву (напр., `YubiKey 5C` або `MacBook Touch ID`)
4. Дотримуйтесь підказки вашого браузера для реєстрації ключа

Ви можете додати кілька ключів безпеки до свого облікового запису для резервування.

### Використання ключів безпеки

Ключі безпеки можуть використовуватися для:

- **Верифікації sudo-режиму** — підтвердження чутливих дій без введення TOTP-коду
- **Відновлення довіри сесії** — повторна верифікація недовіреної сесії після увімкнення 2FA

Коли з'являється запит на верифікацію, просто оберіть опцію ключа безпеки та дотримуйтесь інструкцій браузера.

### Видалення ключа безпеки

Щоб видалити ключ безпеки, перейдіть до **Profile → Security**, знайдіть ключ, який хочете видалити, та натисніть **Remove**. Видалення ключа безпеки потребує активного **sudo-режиму**.

:::tip
Для максимальної безпеки облікового запису використовуйте ключі безпеки **разом з** двофакторною автентифікацією на основі TOTP. Це дає вам кілька методів верифікації та зменшує ризик блокування.
:::

## Управління сесіями

fibe.gg дозволяє моніторити та контролювати всі активні сесії вашого облікового запису, щоб ви завжди знали, де ви авторизовані.

### Термін дії сесій

Сесії дійсні протягом **30 днів** з моменту останньої активності. Після цього вам потрібно буде авторизуватися знову.

### Перегляд активних сесій

Перейдіть до **Profile → Security**, щоб побачити список усіх ваших активних сесій. Для кожної сесії ви можете бачити:

- **IP-адресу**, з якої сесія була створена
- **Браузер та операційну систему**
- **Мітку часу останньої активності**
- Чи є сесія вашою **поточною сесією**

### Відкликання сесій

- **Відкликання однієї сесії**: Натисніть кнопку відкликання поруч з будь-якою сесією, щоб негайно її завершити. Цьому пристрою потрібно буде авторизуватися знову.
- **Відкликання всіх інших сесій**: Натисніть **Revoke All Other Sessions**, щоб завершити кожну сесію, крім поточної. Ця дія потребує **sudo-режиму**.

### Довіра сесій та 2FA

Якщо у вас увімкнено 2FA, сесії повинні бути **довіреними** перед тим, як ви зможете виконувати чутливі дії. Сесія стає довіреною, коли ви підтверджуєте її одним з наступних способів:

- **TOTP-код** з вашого додатку автентифікації
- **Код відновлення**
- **Ключ безпеки**

Довіру можна відновити в будь-який час через запит верифікації, що з'являється при спробі виконати захищену дію.

## Sudo-режим

Деякі чутливі дії у fibe.gg потребують повторної верифікації вашої особи, навіть якщо ви вже авторизовані. Це називається **sudo-режим** — короткочасний підвищений стан, який підтверджує, що саме ви виконуєте дію.

### Як це працює

Коли ви намагаєтесь виконати чутливу дію, fibe.gg запропонує вам підтвердити свою особу. Після підтвердження sudo-режим залишається активним протягом **15 хвилин**, під час яких ви можете виконувати додаткові чутливі дії без повторного запиту.

### Дії, що потребують sudo-режиму

- Створення або знищення **API-ключів**
- Створення, розкриття або знищення **секретів**
- Створення або знищення **кінцевих точок вебхуків**
- Розкриття секретів підпису вебхуків
- Відкликання всіх інших **сесій**
- Додавання або видалення **ключів безпеки**

### Вхід у sudo-режим

Ви можете підтвердити свою особу будь-яким з наступних методів:

- Ввести **TOTP-код** з вашого додатку автентифікації
- Ввести **код відновлення** (одноразовий)
- Автентифікуватися за допомогою **ключа безпеки**

Після верифікації вас повернуть до дії, яку ви намагалися виконати.

## API-ключі

API-ключі дозволяють програмну автентифікацію — для CI/CD конвеєрів, MCP-сервера або будь-яких власних інструментів.

### Створення API-ключа

1. Перейдіть до **Profile → API Keys**
2. Натисніть **New API Key**
3. Дайте йому описову мітку (напр., `GitHub Actions CI`)
4. Оберіть необхідні **scopes**
5. Скопіюйте ваш ключ — він буде показаний лише один раз

:::caution
Зберігайте ваш ключ надійно. fibe.gg показує повний токен лише під час створення.
:::

### Scopes

API-ключі мають обмежений доступ — вони надають доступ лише до того, що ви явно дозволили:

| Scope | Доступ |
|-------|--------|
| `playrooms:read` | Перегляд списку та деталей Playroom |
| `playrooms:write` | Створення Playroom |
| `playrooms:delete` | Видалення Playroom |
| `playspecs:read` | Перегляд списку та деталей Playspec |
| `playspecs:write` | Створення та оновлення Playspec |
| `playspecs:delete` | Видалення Playspec |
| `playgrounds:read` | Перегляд списку та деталей Playground |
| `playgrounds:write` | Створення, продовження та перестворення Playground |
| `playgrounds:delete` | Знищення Playground |
| `playzones:read` | Перегляд списку та деталей Playzone |
| `playzones:write` | Створення Playzone |
| `import_templates:read` | Перегляд та пошук шаблонів |
| `import_templates:write` | Створення та управління шаблонами |
| `genies:read` | Перегляд списку та деталей Генії та їхніх даних |
| `genies:write` | Управління даними Генії |
| `webhooks:read` | Перегляд кінцевих точок вебхуків та доставок |
| `webhooks:write` | Створення та оновлення кінцевих точок вебхуків |
| `webhooks:delete` | Видалення кінцевих точок вебхуків |
| `mcp:access` | Підключення до MCP-сервера |
| `repos:write` | Створення репозиторіїв та надсилання файлів |

### Життєвий цикл ключа

- **Термін дії**: Встановіть необов'язкову дату закінчення терміну дії
- **Ротація**: Відкликайте та повторно створюйте ключі в будь-який час з вашого профілю
- **Останнє використання**: Відстежуйте, коли ключ використовувався востаннє, для аудиту безпеки

## Сховище секретів

Сховище секретів дозволяє безпечно зберігати чутливі облікові дані. Це особливо корисно для безпечного надання облікових даних AI Геніям, підключеним через MCP (Model Context Protocol).

### Управління секретами

Секрети зберігаються як пари ключ-значення з необов'язковим описом.

1. Перейдіть до **Profile → Advanced Settings** (або знайдіть виділений розділ **Secrets**).
2. Натисніть **New Secret**.
3. Вкажіть унікальний буквено-цифровий **ключ** (напр., `OPENAI_API_KEY`).
4. Вкажіть чутливе **значення**.
5. Необов'язково додайте опис.

:::important
Значення шифруються у стані спокою. Будьте обережні щодо того, хто або які Генії мають доступ до отримання ваших секретів. AI Генії з відповідними MCP-інструментами можуть автоматично отримувати ці значення.
:::

## Вебхуки

Вебхуки дозволяють зовнішнім сервісам отримувати сповіщення в реальному часі про події на fibe.gg.

### Підтримувані події

| Подія | Коли спрацьовує |
|-------|----------------|
| `playground.created` | Playground успішно створено |
| `playground.started` | Playground почав працювати |
| `playground.stopped` | Playground зупинено |
| `playground.destroyed` | Playground знищено |
| `playground.error` | Playground перейшов у стан помилки |
| `playground.expiring` | Playground скоро закінчиться |
| `playground.recreated` | Playground перестворено з нуля |
| `playroom.created` | Новий Playroom зареєстровано |
| `playroom.deleted` | Playroom видалено |
| `playspec.created` | Новий Playspec створено |
| `playspec.deleted` | Playspec видалено |

### Створення кінцевої точки вебхуку

1. Перейдіть до **Profile → Webhooks**
2. Натисніть **New Webhook**
3. Введіть цільову **URL-адресу** (повинна бути HTTPS)
4. Оберіть **події**, які хочете отримувати
5. Необов'язково встановіть **секрет** для підпису запитів
6. Натисніть **Save**

### Формат навантаження

Усі події вебхуків доставляються як HTTP POST-запити:

```json
{
  "event": "playground.created",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "id": 123,
    "name": "My Playground",
    "status": "running"
  }
}
```

### Перевірка підписів

Якщо ви встановили секрет, fibe.gg підписує кожен запит заголовком `X-Playground-Signature`:

```
X-Playground-Signature: sha256=abc123...
```

Перевірте його у вашому обробнику, обчисливши `HMAC-SHA256(secret, raw_body)` та порівнявши результати.

### Логи доставки

Кожна доставка вебхуку записується в лог. Перейдіть до кінцевої точки вебхуку для перегляду:
- **Статусу відповіді** для кожної доставки
- **Заголовків та тіла запиту/відповіді**
- **Причини невдачі**, якщо доставка не вдалася
- **Історії повторних спроб** для невдалих доставок

:::tip Тестування
Використовуйте **Test Webhook** для надсилання зразка події `playground.created` до вашої кінцевої точки та перевірки її коректної роботи.
:::
