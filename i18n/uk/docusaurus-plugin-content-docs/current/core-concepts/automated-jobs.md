# Автоматизовані завдання (CI та Muti)

fibe.gg виходить за межі простих середовищ, впроваджуючи **автоматизовані завдання AI-виправлення**. Використовуючи ваших [збережених Генії](/genies/overview) та інфраструктуру [режиму Job](/core-concepts/job-mode), ви можете налаштувати Playspec для автоматичного створення середовищ, які виявляють, аналізують та виправляють проблеми з кодом.

Доступні два основних режими автоматизованих завдань: **Playspec CI-Job** та **Playspec Muti-Job**.

## 1. Playspec CI-Job (безперервна інтеграція з виправленням)

Коли CI-конвеєр або набір тестів завершується невдало, зазвичай розробнику потрібно досліджувати логи та надсилати виправлення. **Playspec CI-Job** автоматизує цей процес, створюючи Job Playground, запускаючи тести та залучаючи AI Генія для негайного пропонування та впровадження виправлень.

### Як це працює:
- **Тригер**: Завдання запускається (напр., через вебхук або API при пуші гілки).
- **Виконання**: З Playspec створюється Job Playground.
- **Ін'єкція Генія**: Налаштований AI Геній запускає набір тестів.
- **Аналіз і виправлення**: Якщо виникають невдачі, Геній отримує вивід тестів, аналізує першопричину та використовує свої MCP-інструменти для модифікації вихідного коду.
- **Верифікація**: Геній повторно запускає тести для перевірки виправлення і, залежно від конфігурації, може надіслати зміни назад у репозиторій.

## 2. Playspec Muti-Job (виправлення мутаційного тестування)

Мутаційне тестування вводить навмисні помилки (мутації) у ваш код для перевірки того, чи ваш набір тестів їх виявляє. "Вижила мутація" означає, що ваші тести неповні. **Playspec Muti-Job** автоматизує процес написання нових тестів для знищення мутацій, що вижили.

### Як це працює:
- **Тригер**: Запускається періодично або за запитом.
- **Виконання**: Середовище генерує список мутацій за допомогою інструментів на кшталт Mutmut або спеціалізованих Muti-фреймворків.
- **Ін'єкція Генія**: AI Геній переглядає різниці мутацій, що вижили.
- **Аналіз і виправлення**: Геній пише нові, всебічні тестові випадки, спеціально розроблені для виявлення зміненої поведінки ("мутанта").
- **Верифікація**: Набір тестів повторно запускається проти мутанта. Якщо мутант знищено, новий тест надсилається в репозиторій.

## Огляд конфігурації

Автоматизовані завдання налаштовуються безпосередньо в Playspec. Ви вказуєте:
- Чи увімкнено функцію.
- Який **збережений Геній** повинен обробляти завдання.
- **Playzone** репозиторію та цільову **гілку**.
- Власний **шаблон промпту** для надання Генію відповідного контексту, такого як логи збірки або різниці мутацій.
- Максимальну кількість спроб повтору.

Після налаштування ці завдання значно зменшують ручні витрати на виправлення зламаних збірок та покращення покриття тестами, використовуючи справжню потужність ваших автономних AI Генії.
