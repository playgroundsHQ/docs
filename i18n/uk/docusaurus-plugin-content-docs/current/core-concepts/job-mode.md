# Режим Job

**Режим Job** дозволяє запускати Playground як безголовий, одноразовий виконувач завдань замість інтерактивного середовища.

Замість довготривалого робочого простору для розробки, Playground у режимі Job:
1. Запускає контейнери
2. Виконує визначений "відстежуваний" сервіс до завершення
3. Фіксує результат
4. Самознищується

Це ідеально підходить для: запуску наборів тестів, міграцій даних, конвеєрів збірки або будь-яких автоматизованих завдань, що не потребують участі людини.

## Увімкнення режиму Job

Встановіть `job_mode: true` при створенні Playground через API:

```bash
POST /api/playgrounds
{
  "playground": {
    "playspec_id": 42,
    "playroom_id": 1,
    "job_mode": true,
    "watched_services": ["tests"]
  }
}
```

## Відстежувані сервіси

Масив `watched_services` містить список контейнерних сервісів, які відстежує режим Job. Коли **всі** відстежувані сервіси завершують роботу, Playground:

1. Позначається як завершений
2. Результат завдання (код виходу + останні логи) фіксується та зберігається
3. Playground автоматично знищується — без витрат на простій

Якщо `watched_services` не вказано, відстежуються всі динамічні сервіси.

## Короткий TTL

Playground у режимі Job автоматично використовують значно коротший час життя (TTL) порівняно з інтерактивними Playground — зазвичай 1 година замість стандартних 8 годин. Це захищає від некерованих завдань.

## Доступ до результатів

Після завершення результат завдання доступний через API:

```bash
GET /api/playgrounds/:id/job_result
```

Відповідь:
```json
{
  "exit_code": 0,
  "completed_at": "2024-01-15T10:35:00Z",
  "tail_logs": "...\nAll 42 tests passed."
}
```

## Варіанти використання

| Варіант використання | Приклад |
|---------------------|---------|
| Запуск тестів | Запуск `rspec` на функціональній гілці |
| Міграція бази даних | Запуск `rails db:migrate` з паритетом production |
| Конвеєр збірки | Збірка та надсилання Docker-образу |
| Генерація документації | Запуск `bundle exec yard` та завантаження документації |
| Якість коду | Запуск лінтерів, сканерів безпеки, звітів покриття |
