# Збережені агенти

Збережені агенти дозволяють **назавжди зареєструвати облікові дані та конфігурації вашого AI провайдера** безпосередньо у fibe.gg. Після збереження ви можете:

- **Підключати** їх до будь-якого [Майданчика](/core-concepts/playground), автоматично інжектуючи вашу конфігурацію в sidecar-контейнер агента
- **Запускати AI чат загального призначення** на будь-якій з ваших [Баз](/core-concepts/playroom) одним кліком — без створення Майданчика
- **Повторно використовувати** їх у різних Майданчиках і чат-сесіях без постійного налаштування

## Налаштування агентів

Агенти у fibe.gg гнучко налаштовуються. За допомогою **Майстра налаштування агента** ви можете:

- **Встановити власний аватар:** Завантажити кастомні зображення профілю або згенерувати унікальні SVG-аватари роботів через Active Storage.
- **Визначити системні промпти:** Надати специфічні інструкції або контекст, якого агент повинен дотримуватися у всіх своїх взаємодіях.
- **Підключити навички (Skills):** Розширити можливості агента за допомогою кастомних навичок, адаптованих для ваших конкретних середовищ.
- **Інтегрувати Custom MCPs:** Підключити власні сервери Model Context Protocol, щоб дати агентам доступ до ваших внутрішніх інструментів та даних.

## Підтримувані типи автентифікації

Оскільки ви надаєте облікові дані, ви маєте повний контроль над квотами та лімітами вашого провайдера:

| Тип автентифікації | Формат облікових даних |
|---|---|
| **OAuth / JSON** | Безпечне введення JSON метаданих |
| **Ручний токен** | Стандартний API токен або OAuth токен |
| **Device Code / Ключ** | Стандартний `auth.json` або API ключ |

## Створення агента

1. Перейдіть до **Агенти** на бічній панелі
2. Натисніть **Новий агент**
3. Скористайтеся **Майстром налаштування агента**, щоб визначити поведінку агента
4. Введіть **Назву**
5. Оберіть **Тип провайдера**
6. Налаштуйте **Аватар**, **Системний промпт**, та опціонально **Custom MCP**
7. Натисніть **Створити агента**

:::info Квота
Кожен обліковий запис може зберігати до **10 агентів** — наприклад, окремі агенти для різних проєктів або команд.
:::

## Статуси агента

| Статус | Значення |
|---|---|
| **Очікує** (Pending) | Створено, але ще не автентифіковано |
| **Автентифіковано** (Authenticated) | Облікові дані збережено, агент готовий до роботи |
| **Закінчився** (Expired) | Термін дії облікових даних закінчився |
| **Відкликано** (Revoked) | Облікові дані відкликано вручну |

Агент **може використовуватися**, якщо його статус `authenticated` і необов'язковий термін дії ще не настав.
