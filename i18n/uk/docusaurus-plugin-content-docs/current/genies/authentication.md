# Автентифікація

Після створення агент знаходиться у статусі **Очікує**. Натисніть **Автентифікувати** на сторінці деталей агента, щоб підключити ваш обліковий запис провайдера. Кожен провайдер має свій потік автентифікації — ви використовуєте ті самі облікові дані, що й для CLI-інструментів провайдера.

## Потік OAuth / JSON

1. Натисніть **Відкрити URL автентифікації** → увійдіть через ваш обліковий запис провайдера
2. Вставте повний JSON-вміст облікових даних у текстове поле
3. Натисніть **Надіслати**

Це ті самі облікові дані, які CLI-інструмент провайдера використовує локально — ваше використання буде зараховуватись до квоти вашої підписки.

:::tip
JSON перевіряється перед збереженням. Невалідний JSON відхиляється з чітким повідомленням про помилку.
:::

## Потік Ручного токена

1. Вставте ваш OAuth токен провайдера або стандартний токен у текстове поле
2. Натисніть **Надіслати**

Токен безпечно зберігається у контейнері агента. Усе використання працює в межах лімітів вашої підписки.

## Потік API Ключа / Device Code

1. Завершіть авторизацію пристрою на сайті провайдера (якщо застосовується)
2. Вставте вміст `auth.json` (або API ключ) у текстове поле
3. Натисніть **Надіслати**

Якщо вставлений вміст є валідним JSON, він зберігається як є. Якщо це API ключ, він автоматично форматується. Ваш тарифний план визначає доступні моделі та ліміти швидкості.

## Повторна автентифікація, відкликання та видалення

| Дія | Що відбувається |
|---|---|
| **Повторна автентифікація** | Замінює існуючі облікові дані новими. Доступно, коли агент вже автентифікований. |
| **Відкликати** | Стирає збережені облікові дані та встановлює статус `revoked`. Агент стає непридатним для використання, але запис залишається. |
| **Видалити** | Повністю видаляє агента та всі пов'язані чат-сесії. |

Усі деструктивні дії потребують підтвердження.

## Безпека облікових даних

- Усі облікові дані **зашифровані** через Active Record encryption
- Максимальний розмір облікових даних: **64 КБ**
- Відкликання агента повністю стирає облікові дані
