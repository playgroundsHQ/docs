# Портативність даних

Портативність даних дозволяє **експортувати** та **імпортувати** конфігурацію вашого облікового запису fibe.gg. Використовуйте це для створення резервних копій, міграції між середовищами або перенесення налаштувань на інший обліковий запис.

## Дані для експорту

Наступні сутності можуть бути включені в експорт:

| Сутність | Опис |
|----------|------|
| **Playrooms** | Ваші налаштовані віддалені хости та їхні параметри |
| **Playzones** | Підключені репозиторії та їхня конфігурація синхронізації |
| **Playspecs** | Шаблони середовищ, включаючи визначення сервісів |
| **Генії** | Збережені конфігурації та облікові дані AI-провайдерів |
| **Playgrounds** | Запущені або зупинені екземпляри середовищ |
| **Шаблони** | Багаторазові шаблони середовищ та їхні версії |
| **API-ключі** | Ваші видані API-ключі |
| **Секрети** | Зашифровані секрети зі сховища секретів |
| **Webhook-ендпоінти** | Налаштовані слухачі webhook |

## Експорт ваших даних

1. Перейдіть до **Профіль → Розширені налаштування → Портативність даних**
2. Оберіть сутності, які хочете експортувати
3. Опціонально увімкніть **сповіщення електронною поштою**, щоб отримати повідомлення про готовність експорту
4. Натисніть **Експортувати**
5. Після завершення завантажте файл з вкладки **Історія**

:::info Доступність експорту
Завершені експорти доступні для завантаження протягом **24 годин**. Після цього файл автоматично видаляється.
:::

## Імпорт даних

1. Завантажте файл `.jsonl` або `.jsonl.gz`
2. Оберіть, які сутності імпортувати
3. Виберіть **стратегію конфліктів** (див. нижче)
4. Натисніть **Імпортувати**
5. Відстежуйте прогрес на вкладці **Історія**

:::tip
Ви можете імпортувати дані, експортовані з будь-якого середовища fibe.gg, що спрощує перенесення конфігурацій між середовищами staging та production.
:::

## Стратегії конфліктів

При імпорті даних, що перетинаються з існуючими записами, ви обираєте спосіб обробки конфліктів:

| Стратегія | Поведінка |
|-----------|-----------|
| **Злиття** | Оновити існуючі записи імпортованими даними |
| **Пропуск** | Зберегти існуючі записи та імпортувати лише нові |

## Історія та статус

Вкладка **Історія** надає перегляд усіх ваших трансферів даних у реальному часі:

- Відстежуйте прогрес поточних експортів та імпортів
- Завантажуйте завершені файли експорту
- Переглядайте деталі та статус минулих трансферів

## Відкат

Якщо імпорт дає неочікувані результати, завершені імпорти можна **відкотити**, відновивши ваші дані до стану до імпорту.

## Обмеження

| Обмеження | Значення |
|-----------|----------|
| Одночасні трансфери | **1** активний трансфер одночасно |
| Максимальний розмір файлу | **50 МБ** |
| Максимальна кількість записів | **5 000** за трансфер |
