# Обмеження швидкості

Для забезпечення стабільності платформи та справедливого використання ресурсів fibe.gg впроваджує комплексне обмеження швидкості на базі **Rack Attack**.

## Глобальні обмеження

Платформа застосовує стандартні обмеження швидкості для всіх користувачів:

| Область | Обмеження | Період |
|---------|-----------|--------|
| **Веб-інтерфейс** (за IP) | 5 000 запитів | 1 година |
| **API** (за гравцем) | 5 000 запитів | 1 година |
| **API** (за API-ключем) | 5 000 запитів | 1 година |
| **MCP** (за гравцем) | 5 000 запитів | 1 година |

## Індивідуальні ліміти для гравця

Якщо ваш проєкт потребує вищої пропускної здатності, адміністратори можуть налаштувати ліміти для вашого облікового запису з панелі адміністратора. Ці перевизначення застосовуються до всіх запитів, пов'язаних з вашим ID гравця, та мають пріоритет над глобальними стандартними значеннями.

## Обмеження API-ключа

Кожен API-ключ може мати власне обмеження швидкості, що дозволяє ізолювати різні інтеграції. Коли використовується API-ключ, його специфічне обмеження має пріоритет над глобальним лімітом гравця.

| Функція | Опис |
|---------|------|
| **Погодинні вікна** | Усі обмеження використовують 1-годинне ковзне вікно для передбачуваної та щедрої пропускної здатності. |
| **На базі Redis** | Стан обмежень зберігається в керованому кластері Redis, забезпечуючи точність на всіх репліках веб-контейнерів. |

## Заголовки відповіді

Кожна відповідь API та MCP містить заголовки обмеження швидкості для моніторингу вашої квоти:

| Заголовок | Опис |
|-----------|------|
| `X-RateLimit-Limit` | Максимальна кількість запитів у поточному вікні. |
| `X-RateLimit-Remaining` | Залишок запитів у поточному вікні. |
| `X-RateLimit-Reset` | Секунди до скидання поточного вікна. |

## Обробка помилок обмеження швидкості

Коли ви перевищуєте ліміт, платформа повертає код статусу `429 Too Many Requests`. Відповідь містить заголовок `Retry-After`, що вказує кількість секунд очікування перед повторною спробою.

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded. Retry after 1234 seconds."
  }
}
```

:::tip
Для програмних інтеграцій завжди реалізовуйте експоненціальне відступлення для коректної обробки обмежень швидкості. Використовуйте заголовок `X-RateLimit-Remaining` для проактивного сповільнення до досягнення ліміту.
:::
